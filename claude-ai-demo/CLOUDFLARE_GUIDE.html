Key Concepts Before You Start

Cloudflare Pages — hosts your React frontend (like Netlify/Vercel). Put JS files in the functions/ folder and they become serverless API endpoints automatically.
Cloudflare Workers AI — run AI models (Llama, Mistral etc.) on Cloudflare's servers for free. No API key needed — just env.AI.run("model-name", {...}).
Cloudflare R2 — object storage (like AWS S3 but free for 10GB). Store and retrieve files from your Workers using env.BUCKET_NAME.put() and .get().
Bindings — how your Worker code accesses Cloudflare services. Configured in wrangler.jsonc for local dev and in the Cloudflare Dashboard for production.

Prompt Engineering Basics

Every AI API call has two parts:

🟣 System Prompt — tells the AI WHO it is and WHAT FORMAT to use
You are an expert HR assistant.
Always respond with valid JSON only — no extra text.
Use this structure: { "name": "...", "skills": [...] }
🔵 User Message — context injection (real data from the user)
Please summarise the following CV:

Jane Smith, Senior Engineer at TechCorp...
[CV text injected here dynamically]
🟢 AI Response — structured JSON we parse in React
{ "name": "Jane Smith", "title": "Senior Engineer", "skills": ["React", "Node.js"] }
ConceptWhat it meansWhere in code
System promptAI's role + output formatfunctions/api/*.js
Context injectionInject user data into the promptuserMessage = `...${cvText}`
Structured outputAsk AI to reply in JSONJSON.parse(aiResponse.response)

Install Dependencies & Login

Open a terminal inside the project folder:

cd claude-ai-demo
npm install

Login to Cloudflare (opens a browser window):

npx wrangler login
  • ✅ A browser window will open — click Allow
  • ✅ You'll see Successfully logged in in the terminal
3

Create the R2 Bucket

R2 is Cloudflare's file storage. Run this once to create the bucket:

npx wrangler r2 bucket create cv-storage

Verify it was created:

npx wrangler r2 bucket list
✅ Expected output: You should see cv-storage in the list.
💡 What is a bucket? Think of it like a folder in the cloud. You store files in it and retrieve them using a key (filename). The first 10GB per month is free.
4

Run Locally

You need two terminals running at the same time:

Terminal 1 — React frontend (Vite)

npm run dev

Terminal 2 — Cloudflare Worker (handles /api routes)

npx wrangler pages dev --proxy 5173

Then open your browser at:

http://localhost:8788
⚠️ Important: Use port 8788 (not 5173) when testing locally. Port 8788 is the Cloudflare Worker, which handles the /api routes. Port 5173 is Vite only and won't have the API.
💡 Local vs Deployed: When running locally, R2 files are stored in .wrangler/state/ on your machine — not in the real Cloudflare R2 bucket. To see files in the dashboard, you must test on the deployed URL.
5

Deploy to Cloudflare Pages

Build the React app

npm run build

Deploy to Cloudflare

npx wrangler pages deploy dist
✅ You'll get a live URL like:
https://claude-ai-demo.pages.dev
⚠️ Every time you change code, run npm run build then npx wrangler pages deploy dist again to update the live site.
6

Add Bindings in Cloudflare Dashboard

Bindings connect your deployed Workers to Cloudflare services. You must add them manually in the dashboard — wrangler.jsonc only works for local development.

Go to:

dash.cloudflare.com → Workers & Pages → claude-ai-demo → Settings → Functions

Add Workers AI Binding

  • ➡️ Scroll to Workers AI Bindings
  • ➡️ Click Add binding
  • ➡️ Variable name: AI
  • ➡️ Click Save

Add R2 Bucket Binding

  • ➡️ Scroll to R2 bucket bindings
  • ➡️ Click Add binding
  • ➡️ Variable name: CV_BUCKET
  • ➡️ Bucket: select cv-storage
  • ➡️ Click Save
💡 Why do we need this? In code we write env.AI and env.CV_BUCKET. The binding is what connects those names to the actual Cloudflare services in production.

Redeploy after adding bindings

npm run build
npx wrangler pages deploy dist

How the Code Works — Key Files Explained

functions/api/upload-cv.js

// 1. Read the uploaded file from the form
const file = formData.get("file");
const extractedText = formData.get("text"); // extracted by PDF.js in browser

// 2. Store in R2 — env.CV_BUCKET is the binding
await env.CV_BUCKET.put(key, file.stream());        // original file
await env.CV_BUCKET.put(`${key}.txt`, extractedText); // extracted text

functions/api/ask-cv.js

// 1. Read the CV text back from R2
const object = await env.CV_BUCKET.get(`${key}.txt`);
const cvText = await object.text();

// 2. Inject CV text into the prompt (context injection)
const userMessage = `Here is the candidate's CV:\n${cvText}\n\nQuestion: ${question}`;

// 3. Call Cloudflare Workers AI — completely free!
const aiResponse = await env.AI.run("@cf/meta/llama-3.1-8b-instruct", {
  messages: [
    { role: "system", content: systemPrompt },
    { role: "user",   content: userMessage  },
  ]
});

src/pages/CVSummariser.jsx (frontend)

// 1. Extract text from PDF in the browser (PDF.js)
const pdf = await pdfjsLib.getDocument({ data: arrayBuffer }).promise;
const pageText = content.items.map(item => item.str).join(' ');

// 2. Upload file + text to the Worker
const formData = new FormData();
formData.append('file', file);
formData.append('text', extractedText);
await fetch('/api/upload-cv', { method: 'POST', body: formData });

// 3. Ask the AI a question

  method: 'POST',

});

Troubleshooting

Problem Cause Fix
Network error on upload Wrangler not running Run npx wrangler pages dev --proxy 5173 in Terminal 2
File uploaded but not in R2 dashboard Testing on localhost Test on the deployed URL — local files go to .wrangler/state/
AI not responding on deployed site Workers AI binding missing Add AI binding in Dashboard → Settings → Functions
Upload fails on deployed site R2 binding missing Add CV_BUCKET → cv-storage binding in Dashboard
dist folder not found Forgot to build Run npm run build before deploying
Changes not showing on live site Old deployment Run npm run build && npx wrangler pages deploy dist
