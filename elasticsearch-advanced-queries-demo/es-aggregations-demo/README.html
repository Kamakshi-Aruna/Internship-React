ES Aggregations — Beginner Guide

Learn how Elasticsearch aggregations work: Terms for faceted counts, Date Histogram for trend charts, and Nested aggregations for multi-level drill-down. All exposed via a REST API.

Elasticsearch 8.x Node.js + Express REST API Beginner Friendly

1 What is an Aggregation?

Simple analogy: A regular ES search says "give me documents that match this query." An aggregation says "give me statistics about those documents — counts, averages, time buckets, etc."

In SQL terms:

  • Regular search → SELECT * FROM candidates WHERE name LIKE '%React%'
  • Aggregation → SELECT location, COUNT(*) FROM candidates GROUP BY location

The key pattern: use "size": 0 when you only want aggregation results (not raw documents).

// Basic aggregation shape
GET candidates/_search
{
  "size": 0,        // skip raw documents — only want stats
  "aggs": {
    "my_agg_name": {  // you pick the name
      "terms": {       // aggregation type
        "field": "location"
      }
    }
  }
}

2 Terms Aggregation — Faceted Counts

The most common aggregation. Counts how many documents fall into each unique value of a keyword field. Perfect for filter sidebars, tag clouds, and pie charts.

SQL: SELECT status, COUNT(*) FROM candidates GROUP BY status

2a. Count by Location

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "by_location": {
      "terms": {
        "field": "location",  // must be a keyword field
        "size": 20            // top 20 locations
      }
    }
  }
}

Response shape: aggregations.by_location.buckets → array of { key: "San Francisco", doc_count: 12 }

2b. Count by Skill (keyword sub-field)

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "top_skills": {
      "terms": {
        "field": "skills.keyword",  // .keyword = exact match sub-field
        "size": 10
      }
    }
  }
}
Why .keyword? Text fields (like skills) are analyzed (tokenized) for full-text search. To aggregate on exact values you need the raw keyword sub-field. In the mapping: "skills": { "type": "text", "fields": { "keyword": { "type": "keyword" } } }

2c. All Facets in One Query

You can define multiple top-level aggregations — ES computes them all in one pass:

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "skills_facet":   { "terms": { "field": "skills.keyword", "size": 10 } },
    "location_facet": { "terms": { "field": "location" } },
    "status_facet":   { "terms": { "field": "status" } }
  }
}

3 Date Histogram Aggregation — Trend Charts

Groups documents into equal-width time buckets. Great for line charts showing how activity changes over time.

SQL: SELECT DATE_FORMAT(applied_at, '%Y-%m'), COUNT(*) FROM candidates GROUP BY 1

3a. Applications per Month

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "applications_trend": {
      "date_histogram": {
        "field":             "applied_at",  // must be a date field
        "calendar_interval": "month",       // day | week | month | quarter | year
        "format":           "yyyy-MM",     // human-readable bucket keys
        "min_doc_count":     0              // include months with 0 applications
      }
    }
  }
}

Response: { key_as_string: "2024-03", doc_count: 5 } — one bucket per month.

3b. Extended Bounds (show empty months)

"date_histogram": {
  "field":             "applied_at",
  "calendar_interval": "month",
  "format":           "yyyy-MM",
  "min_doc_count":     0,           // allow empty buckets
  "extended_bounds": {
    "min": "now-1y/M",             // start of last year
    "max": "now/M"                // current month
  }
}
Tip: Without extended_bounds, ES only returns months that have data. With it, you get a complete timeline — important for line charts that shouldn't have gaps.

4 Nested (Multi-Level) Aggregations — Drill-Down

Put an aggs block inside a bucket aggregation to drill down further. ES computes the inner aggregation separately for each bucket of the outer one.

4a. Skills per Location (2 levels deep)

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "by_location": {
      "terms": { "field": "location", "size": 10 },

      "aggs": {                        // ← nested agg lives here
        "top_skills": {
          "terms": { "field": "skills.keyword", "size": 5 }
        }
      }
    }
  }
}
// Result shape:
// by_location.buckets[0].key         → "San Francisco"
// by_location.buckets[0].doc_count   → 8
// by_location.buckets[0].top_skills.buckets → [{ key: "React", doc_count: 5 }, ...]

4b. Metric Sub-Aggregations (avg, min, max)

Metric aggregations compute a single number (not buckets) from the matched documents:

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "by_status": {
      "terms": { "field": "status" },

      "aggs": {
        "avg_exp": { "avg": { "field": "experience_years" } },
        "min_exp": { "min": { "field": "experience_years" } },
        "max_exp": { "max": { "field": "experience_years" } }
      }
    }
  }
}

4c. Date → Location (3 levels: month → location → count)

GET candidates/_search
{
  "size": 0,
  "aggs": {
    "by_month": {
      "date_histogram": {
        "field": "applied_at", "calendar_interval": "month", "format": "yyyy-MM"
      },
      "aggs": {
        "by_location": {
          "terms": { "field": "location", "size": 5 }
        }
      }
    }
  }
}

5 Aggregations + Query Filter

Add a "query" block to limit which documents go into the aggregation. The aggregation then runs only on the matching subset.

// Top skills — only among ACTIVE candidates
GET candidates/_search
{
  "size": 0,
  "query": {
    "term": { "status": "active" }
  },
  "aggs": {
    "active_skills": {
      "terms": { "field": "skills.keyword", "size": 10 }
    }
  }
}

7 Quick Reference

Aggregation Types Covered

Type Keyword Use For
Bucket terms Group by keyword value (status, location)
Bucket date_histogram Group by time interval (month, week, year)
Metric avg Average of a numeric field
Metric min / max Minimum / maximum of a numeric field
Nested aggs inside aggs Drill-down: compute stats within each bucket

Common Gotchas

  • Use skills.keyword (not skills) for terms agg on text fields
  • Use calendar_interval in ES 8.x (not the old interval)
  • Set "size": 0 when you only need aggregation output
  • The aggregation name ("by_location") is arbitrary — you choose it
  • min_doc_count: 0 + extended_bounds = complete timeline with zero-gaps
