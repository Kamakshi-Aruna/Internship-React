Before You Begin — Key Concepts Explained

AI, LLMs, and Vector Databases in plain English

Concept 1

What is AI (Artificial Intelligence)?

AI means teaching a computer to do things that normally require human intelligence — like understanding language, recognising faces, or making decisions.

Real-life analogy

Think of training a dog. You show it hundreds of examples — "sit", "stay", "fetch" — and it learns patterns. AI is similar: instead of a dog, it's a program. Instead of commands, you feed it millions of examples (text, images, numbers), and it learns the patterns inside that data.

🖼️

Image AI

Trained on photos — learns to say "this is a cat" vs "this is a dog".

🗣️

Voice AI

Trained on audio — learns to turn speech into text (like Siri or Alexa).

💬

Language AI

Trained on text — learns to read, write, and chat like a human. This is what Claude is.

Concept 2

What is an LLM (Large Language Model)?

An LLM is a type of AI that specialises in understanding and generating text. Claude, ChatGPT, and Gemini are all LLMs. The "large" means it was trained on an enormous amount of text — think billions of web pages, books, and articles.

How does it actually work? (Simple version)

An LLM reads a prompt and predicts what the best next word is, then the next, then the next — until it has a full answer. It's like a very sophisticated autocomplete that has read almost all of the internet.

You type: "The sky is..."
→
LLM predicts: "blue"
→
then: "because..."
→
and so on...
💡
Key point for developers: You never see the math inside an LLM. You just send it text (a prompt) and get text back (a completion). The Claude API is simply the interface for doing this from your code.

LLMs vs traditional programs

Traditional Program LLM
Rules You write every rule (if/else) It learns rules from data
Unknown input Crashes or returns error Makes a reasonable guess
Example task Sort a list of numbers Summarise a news article
Concept 3

What is a Vector Database?

This one sounds scary but the idea is simple. LLMs are very powerful but they have a memory limit — you can only send them so much text at once (called the context window). A Vector DB solves the problem of searching huge collections of documents to find only the relevant parts before sending them to the LLM.

Step 1 — What is a "vector"?

A vector is just a list of numbers that represents the meaning of a piece of text. An AI model converts text into these numbers. Similar meanings produce similar numbers.

"The dog barked loudly"
→
[0.12, 0.87, 0.34, 0.56 ...]
"A puppy made a noise"
→
[0.11, 0.85, 0.36, 0.54 ...]

Notice how similar sentences produce similar numbers? That's the magic — the database can find semantically related content, not just keyword matches.

Real-world analogy — the library

Imagine a giant library with 1 million books. You ask: "Find me everything about training puppies."

Old way

Search every page for the words "puppy" and "training". Misses books that say "dog obedience" instead.

Vector DB

Converts your question to a vector, finds books whose vectors are closest in meaning. Finds "puppy training", "dog obedience", and "canine behaviour" — all semantically related.

How it plugs into an LLM app (RAG pattern)

This pattern is called RAG — Retrieval-Augmented Generation. Instead of stuffing 1 million documents into Claude (impossible), you retrieve only the 3-5 most relevant chunks and send those.

User asks a
question
→
Convert to
vector
→
Search
Vector DB
→
Get top 5
relevant chunks
→
Send to
Claude API
→

answer
✅
Popular Vector DBs you may hear about: Pinecone, Weaviate, Qdrant, pgvector (a Postgres extension), and Chroma. For this guide we focus on the Claude API — Vector DBs become relevant when you build a full RAG app later.

Introduction to the Claude API

What it is, why we use it, and how it works

Concept

What is the Anthropic Claude API?

The Claude API lets your app talk to Claude AI — you send it a message, it sends back a smart response. Think of it like texting a genius assistant who can answer questions, write code, summarize documents, and much more.

🤖

Claude

Anthropic's AI model — the brain behind your app.

🔗

API

A bridge: your code sends a request, Claude sends a response.

⚡

Streaming

Get the response word-by-word in real time, not all at once.

🔑

API Key

Your secret password to access the Claude API.

How It Works

The request/response cycle

Every interaction follows this simple flow:

Your App
→
POST /v1/messages
→
Anthropic Server
→
Claude Model
→
Response JSON

You send a messages array (a conversation history) and Claude responds with text. That's it at its core!

Models

Which model should I use?

For this guide we'll use claude-sonnet-4-6 — a great balance of speed and intelligence. Here's the quick reference:

Model Use When Cost
claude-opus-4-6 Complex reasoning, best quality $$$
claude-sonnet-4-6 ✓ General use, this guide $$
claude-haiku-4-5 Simple tasks, fastest, cheapest $

Set Up Your API Key

Get your key and add it to the project safely

Step 2.1 — Get your key

Create an Anthropic API key

  • ✓ Go to console.anthropic.com and sign in
  • ✓ Click API Keys in the left sidebar
  • ✓ Click Create Key, give it a name (e.g. "intern-dev")
  • ✓ Copy the key — it starts with sk-ant-...
🚨
Never commit your API key to Git! If you accidentally push it, rotate it immediately in the console. Anyone with your key can use your account.
Step 2.2 — Create .env file

Store your key in an environment variable

In the root of the project, create a file named .env:

Shell / .env 
# .env  — DO NOT commit this file to Git
ANTHROPIC_API_KEY=sk-ant-your-actual-key-here
💡
Open .gitignore and make sure .env is listed there. If it's missing, add it on its own line.
Step 2.3 — Install the SDK

Add the Anthropic SDK to your project

In your terminal, from the project root:

Terminal 
# Using npm
npm install @anthropic-ai/sdk

# Or yarn
yarn add @anthropic-ai/sdk

Check it worked — you should see @anthropic-ai/sdk inside package.json dependencies.

Step 2.4 — Server side only!

Important: API calls belong on the server

Your API key must never be sent to the browser. React code runs in the browser, so we need a small backend (Node.js / Express) to make the actual API calls. Your React component talks to your server, which then calls Claude.

Browser (React)
→
Your Server (Node)
→
Claude API

Make Your First API Request

Send a message and get a response from Claude

Step 3.1 — Basic Node.js test

Create a quick test script

Create a file called test-claude.js in your project root and run it with Node.js to verify your key works.

JavaScript — test-claude.js 
// test-claude.js
import Anthropic from "@anthropic-ai/sdk";

// The SDK automatically reads ANTHROPIC_API_KEY from your environment
const client = new Anthropic();

const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [
    {
      role: "user",
      content: "Say hello and tell me one fun fact about space!"
    }
  ]
});

// response.content is an array — grab the first text block
console.log(response.content[0].text);
Terminal 
node test-claude.js
✅
If you see Claude's response printed in the terminal, your API key is working!
Step 3.2 — Understanding the response

What does the response look like?

Let's look at the full structure Claude returns:

JSON — example response 
{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "model": "claude-sonnet-4-6",
  "content": [
    {
      "type": "text",                 // could also be "tool_use" etc.
      "text": "Hello! Here's a fun fact..."
    }
  ],
  "stop_reason": "end_turn",       // "end_turn" means Claude finished naturally
  "usage": {
    "input_tokens": 25,            // tokens in your message
    "output_tokens": 86            // tokens in Claude's reply
  }
}

The text you want is always at: response.content[0].text

Step 3.3 — Express endpoint

Create a server route for Claude

Now let's expose this from your Express server so React can call it:

JavaScript — server/routes/claude.js 
import express from "express";
import Anthropic from "@anthropic-ai/sdk";

const router = express.Router();
const client = new Anthropic();  // reads ANTHROPIC_API_KEY automatically

// POST /api/claude/chat
router.post("/chat", async (req, res) => {
  try {
    const { message } = req.body;

    const response = await client.messages.create({
      model: "claude-sonnet-4-6",
      max_tokens: 1024,
      messages: [{ role: "user", content: message }]
    });

    res.json({
      reply: response.content[0].text
    });
  } catch (err) {
    console.error("Claude error:", err.message);
    res.status(500).json({ error: "Failed to get response" });
  }
});

export default router;

Build a React Chat Component

A functional component that talks to your server

Step 4.1 — Component skeleton

Create src/components/ClaudeChat.jsx

Here is a simple but complete chat component. Read through each comment — it explains every part.

JSX — ClaudeChat.jsx 
import { useState } from "react";

export default function ClaudeChat() {
  // 1. State: the current text in the input box
  const [input, setInput] = useState("");

  // 2. State: Claude's reply text
  const [reply, setReply] = useState("");

  // 3. State: show a spinner while waiting
  const [loading, setLoading] = useState(false);

  // 4. Called when user clicks "Ask"
  const handleAsk = async () => {
    if (!input.trim()) return;  // do nothing if empty

    setLoading(true);
    setReply("");

    try {
      // 5. Call YOUR server — never call the Claude API directly from here
      const res = await fetch("/api/claude/chat", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ message: input })
      });

      const data = await res.json();
      setReply(data.reply);

    } catch (err) {
      setReply("Error: could not reach the server.");
    } finally {
      setLoading(false);
    }
  };

  return (
    <div style={{ maxWidth: 600, margin: "40px auto", padding: "0 20px" }}>
      <h2>Ask Claude</h2>

      {/* Input area */}
      <textarea
        rows={4}
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Type your question..."
        style={{ width: "100%", padding: "10px" }}
      />

      {/* Submit button */}
      <button
        onClick={handleAsk}
        disabled={loading}
        style={{ marginTop: "10px", padding: "8px 20px" }}
      >
        {loading ? "Thinking..." : "Ask Claude"}
      </button>

      {/* Reply area — only shown when there's a reply */}
      {reply && (
        <div style={{ marginTop: "20px", padding: "16px", background: "#f5f5f5" }}>
          <strong>Claude:</strong>
          <p>{reply}</p>
        </div>
      )}
    </div>
  );
}
Step 4.2 — Use it in your app

Add the component to App.jsx

JSX — App.jsx 
import ClaudeChat from "./components/ClaudeChat";

export default function App() {
  return (
    <div>
      <h1>My AI App</h1>
      <ClaudeChat />
    </div>
  );
}
Step 4.3 — Common mistakes

Watch out for these beginner pitfalls

  • !
    Calling the Claude API directly from React — this exposes your key in the browser. Always go through your server.
  • !
    Forgetting to await fetch() — you'll get a Promise object instead of the data.
  • !
    Not setting Content-Type header — the server can't parse the request body without it.
  • !
    Using setState inside a stale closure — if you use reply inside an async callback, use setReply(prev => ...) for safety.

Stream AI Responses in Real Time

Show each word as it arrives — no waiting for the full response

Concept

What is streaming and why use it?

Without streaming: you wait ~3–10 seconds and then the whole response pops in at once. With streaming: you see each word typed out as Claude "thinks" it — just like ChatGPT. This massively improves perceived performance and feels much more alive.

💡
Streaming uses Server-Sent Events (SSE) — a simple HTTP protocol where the server pushes small chunks of data one at a time. Your browser reads them as they arrive.
Step 5.1 — Streaming server route

Add a streaming endpoint to your server

JavaScript — server/routes/claude.js (add this route) 
// POST /api/claude/stream
router.post("/stream", async (req, res) => {
  const { message } = req.body;

  // 1. Tell the browser this is a streaming SSE response
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  try {
    // 2. Use client.messages.stream() instead of .create()
    const stream = client.messages.stream({
      model: "claude-sonnet-4-6",
      max_tokens: 1024,
      messages: [{ role: "user", content: message }]
    });

    // 3. For each text chunk, send it to the browser
    stream.on("text", (textDelta) => {
      // SSE format: "data: ...\n\n"
      res.write(`data: ${JSON.stringify({ chunk: textDelta })}\n\n`);
    });

    // 4. When done, send a "done" signal and close the stream
    await stream.finalMessage();
    res.write("data: [DONE]\n\n");
    res.end();

  } catch (err) {
    res.write(`data: ${JSON.stringify({ error: err.message })}\n\n`);
    res.end();
  }
});
Step 5.2 — Streaming React component

Update your component to read the stream

JSX — ClaudeChatStreaming.jsx 
import { useState } from "react";

export default function ClaudeChatStreaming() {
  const [input, setInput] = useState("");
  const [reply, setReply] = useState("");
  const [streaming, setStreaming] = useState(false);

  const handleAsk = async () => {
    if (!input.trim()) return;

    setStreaming(true);
    setReply("");  // clear previous reply

    try {
      // 1. Call the streaming endpoint
      const response = await fetch("/api/claude/stream", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ message: input })
      });

      // 2. Get a reader to read the body as it arrives
      const reader = response.body.getReader();
      const decoder = new TextDecoder();

      // 3. Loop — keep reading until "done"
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        // 4. Decode the chunk and parse each SSE line
        const text = decoder.decode(value);
        const lines = text.split("\n");

        for (const line of lines) {
          if (!line.startsWith("data: ")) continue;

          const payload = line.slice(6);  // strip "data: " prefix
          if (payload === "[DONE]") break;

          const { chunk } = JSON.parse(payload);

          // 5. Append each chunk to the reply — prev => ensures no stale state
          setReply(prev => prev + chunk);
        }
      }

    } catch (err) {
      setReply("Error connecting to server.");
    } finally {
      setStreaming(false);
    }
  };

  return (
    <div style={{ maxWidth: 600, margin: "40px auto", padding: "0 20px" }}>
      <h2>Ask Claude (Streaming)</h2>

      <textarea
        rows={4}
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Ask anything..."
        style={{ width: "100%", padding: "10px" }}
      />

      <button
        onClick={handleAsk}
        disabled={streaming}
        style={{ marginTop: "10px", padding: "8px 20px" }}
      >
        {streaming ? "Streaming..." : "Ask Claude"}
      </button>

      {reply && (
        <div style={{
          marginTop: "20px", padding: "16px",
          background: "#f5f5f5", whiteSpace: "pre-wrap"
        }}>
          <strong>Claude:</strong>
          <p>{reply}</p>
          {/* Blinking cursor while streaming */}
          {streaming && <span style={{ animation: "blink 1s infinite" }}>▌</span>}
        </div>
      )}
    </div>
  );
}
