1.4 Auth Middleware

Client sends Firebase idToken in Authorization: Bearer <token> header. Server uses firebase-admin to verify the token on every protected route and attaches req.user (uid, email) for downstream use.

3.2 Axios Instance with Interceptors

src/api/axiosInstance.js

const instance = axios.create({ baseURL: import.meta.env.VITE_API_URL });

// Request interceptor — attach Firebase token to every request
instance.interceptors.request.use(async (config) => {
  const token = await auth.currentUser?.getIdToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Response interceptor — handle 401, normalize errors
instance.interceptors.response.use(
  (res) => res,
  (error) => {
    if (error.response?.status === 401) { /* redirect to /login */ }
    return Promise.reject(error);
  }
);

3.3 AuthContext

src/contexts/AuthContext.jsx — wraps onAuthStateChanged from Firebase. Exposes: user, loading, login(email, pass), logout(). ProtectedRoute reads from this context.

3.4 Pages

Login.jsx — Ant Design Form, Input, Button. Calls login() from AuthContext. Shows Alert on error, Spin on loading.

Dashboard.jsx — Fetches /api/stats on mount. Stat cards (antd Statistic): Total Candidates, Total Jobs, Hired, Open Jobs. Bar chart (BarChart): Candidates by status. Pie chart (PieChart): Jobs by status.

Candidates.jsx — Fetches /api/candidates on mount. AG Grid table: name, email, phone, status, role, actions. Add/Edit Modal (antd Modal + Form). Delete with antd Popconfirm.

Jobs.jsx — Same pattern as Candidates. Fields: title, department, location, status, openings.

3.5 AppLayout (Sidebar)

Ant Design Layout with Sider, Header, Content. Menu items: Dashboard, Candidates, Jobs. User avatar + logout button in Header. Collapsible sidebar.

3.6 Loading & Error Pattern (used in every page)

const [data, setData] = useState([]);
const [loading, setLoading] = useState(false);
const [error, setError] = useState(null);

useEffect(() => {
  setLoading(true);
  axiosInstance.get('/api/candidates')
    .then(res => setData(res.data))
    .catch(err => setError(err.response?.data?.message || 'Failed to load'))
    .finally(() => setLoading(false));
}, []);

if (loading) return <Spin />;
if (error) return <Alert type="error" message={error} />;

Bonus Learning Module — Elasticsearch

A separate beginner-friendly folder has been created to learn Elasticsearch concepts hands-on using the Talent Tracker candidate data.

Data IndexingNode.js script to bulk-index MongoDB candidates into Elasticsearch
Kibana Dev Tools15 beginner queries — match, term, range, bool, aggregations

How to start

  1. Open elasticsearch-concepts-demo/README.html in your browser — this is the full guide
  2. Run docker compose up -d inside that folder to start Elasticsearch + Kibana
  3. Run node scripts/sample-data.js to seed MongoDB with sample candidates
  4. Run node scripts/index-candidates.js to push data into Elasticsearch
  5. Open http://localhost:5601 → Dev Tools → paste queries from kibana-queries/basic-queries.txt
